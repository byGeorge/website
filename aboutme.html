Why Programming?
As a relatively new science, there's a lot of new things happening. As the hardware improves, we can do more interesting things with the software. I'm fascinated with all the cool new things computers can do, and excited that I seem to have the talent for it.
Languages, Platforms and Services Oh My!
My mother taught me to choose the best tool for the job, and technological tools are like any other sort of tool. Assembly is like a hammer: powerful, direct, and bit clumsy. Some developers hate assembly... but sometimes you just need to smack something with a hammer to get the job done. Rails is like a laser cutter: you need to know what you're doing, but it can make beautiful things. Like any good craftsperson, I'm more fond of some tools than others, but I can use more than one.
What do you Know?
Languages: I'm most proficient in Ruby and Java, but Python was super easy. C and Assembly were more challenging, but very powerful.

Platforms: Rails was challenging at first, but I absolutely love it now. Django was super easy, but limited. Spring was relatively easy to use, but required some imagination to get it to do what I wanted it to do. YAML is my favorite way to get different languages to interact.

Services: AWS is very powerful, and I prefer it to GoDaddy's web hosting (which doesn't support Ruby or Rails). Heroku is fine for testing deployment, but doesn't scale well and the free version takes forever to load.

Databases: MySQL, SQLite, PostgreSQL. I prefer to use an IDE to visualize databases but can program SQL if needed.

Version control: Git. Version control is my favorite thing about modern programming.
